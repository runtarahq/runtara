# runtara-agent-crypto

[![Crates.io](https://img.shields.io/crates/v/runtara-agent-crypto.svg)](https://crates.io/crates/runtara-agent-crypto)
[![Docs.rs](https://docs.rs/runtara-agent-crypto/badge.svg)](https://docs.rs/runtara-agent-crypto)

The crypto agent (SHA-256/512/1, MD5, HMAC-SHA-256/512) as a WebAssembly component. First port from the legacy `runtara-agents` monolith and the reference template for the rest of the migration.

## Capabilities

| Capability | Input schema | Output |
| ---------- | ------------ | ------ |
| `hash`     | `{ data: string \| FileData, algorithm?: "sha256"\|"sha512"\|"sha1"\|"md5", output_format?: "hex"\|"base64" }` | `{ hash, algorithm, format }` |
| `hmac`     | `{ data, secret: string, algorithm?: "hmac-sha256"\|"hmac-sha512", output_format?: "hex"\|"base64" }` | `{ hash, algorithm, format }` |

`FileData` is `{ content: string (base64), filename?: string, mime_type?: string }`. Schema mirrors `crates/runtara-agents/src/agents/crypto.rs` byte-for-byte so A/B parity tests can compare results across engines.

## Building

```bash
cargo component build --release --target wasm32-wasip2 -p runtara-agent-crypto
# or, build every agent component at once:
./scripts/build-agent-components.sh
```

Output: `target/wasm32-wasip1/release/runtara_agent_crypto.wasm` (~150 KB).

## Anatomy

- [`src/lib.rs`](src/lib.rs) — Rust-side typed inputs (`HashInput`, `HmacInput`), `Guest` impl, JSON Schema strings published via `list-capabilities()`.
- [`src/bindings.rs`](src/bindings.rs) — auto-generated by `cargo component`. Regenerated on each build; committed so plain `cargo build` works without the cargo-component CLI.
- [`Cargo.toml`](Cargo.toml) — `crate-type = ["cdylib", "rlib"]` + `[package.metadata.component.target]` pointing at `runtara-agent-wit`'s WIT package.

## Template for new agents

Copy this crate, rename, replace the `Guest::invoke` match table, update `list_capabilities()`, point `[package.metadata.component]` at the new crate name. Add the crate path to the workspace `Cargo.toml` and to `scripts/build-agent-components.sh` will pick it up automatically (the script scans `crates/runtara-agent-*` excluding `wit`/`macro`/`common`).

Once built, drop the `.wasm` into the directory pointed at by `RUNTARA_AGENT_COMPONENTS_DIR` and restart the server.
