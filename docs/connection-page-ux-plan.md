## Conflict resolutions (decided up front)

| # | Conflict | Ruling | Why |
|---|---|---|---|
| C1 | Never-authorized pill: page-composition proposed `info` (pulse) / "Not connected yet"; flows proposed `warning` / `Authorization needed` | **Flows wins**: warning tone, label `Authorization needed` | Flows is the copy authority; warning is honest — workflows referencing the connection fail until authorized. `info` under-signals a broken state. |
| C2 | Dirty-form Reconnect guard: page-composition = disabled button + tooltip; flows = intercept dialog with `Save & Reconnect` | **Flows wins**: intercept dialog, guard scoped to descriptor-param dirtiness only (title/rate-limit/defaults never block) | A disabled primary action is a dead end; the dialog explains and offers the combined path. Scoping uses the exact filter already in `pages/Connection/update-payload.ts:39-41`. |
| C3 | Post-save reconnect detection: page-composition = predictive-only (`patchStripsAuthorization`); flows = predictive ∧ authoritative response status; contract offers optional `authorization_reset` bool | **Flows wins** for v1 (predictive flag fired AND refetched `status === 'REQUIRES_RECONNECTION'`); contract's `authorization_reset` demoted to UX-3 convenience | Response-only would also fire for already-broken connections; predictive-only can lie if the server didn't actually strip. The backend bool is redundant with data already on the wire (`fieldBehaviors[].requiresReauthorization`, `generated/RuntaraRuntimeApi.ts:1511-1515`). |
| C4 | Create-**success** routing: page-composition = land on new edit page; flows = keep list destination | **Flows wins**: success → `/connections` + `toast.success`; abandon/failure → the new connection's edit page (both lenses agree on abandon) | After a successful authorize there is nothing to do on the edit page; the list confirms completion. Abandon needs the edit page because that's where the recovery CTA (`Connect`) lives. |
| C5 | ACTIVE non-OAuth pill label: page-composition = `Active`; flows = `Connected` | **Flows wins**: `Connected` for both | One label, one copy table (flows §0/§7d). |
| C6 | Test button before backend ships: page-composition = hidden; flows = full disabled-state matrix | **Page-composition wins for gating** (hidden, not disabled, until `POST /connections/{id}/test` exists); flows' state matrix applies once it ships | Don't render affordances that cannot work. |
| C7 | Interim `REQUIRES_RECONNECTION` copy (pre-`grantState`): two wordings proposed | **Flows verbatim wins**: `This connection isn't authorized. Your saved credentials are kept — authorize without re-entering them.` button `Connect` | Never asserts expiry/revocation, never asserts first-time — the only copy that is true in both cases. |
| C8 | New status enum value for never-authorized | **Contract wins**: derived flag from `grantState`, **no** `PENDING_AUTHORIZATION` enum | The enum round-trips through DB column, wire serde, list filters, and the callback's hardcoded `"ACTIVE"` write (`crates/runtara-connections/src/service/oauth.rs:248`) — a migration for something `hasAccessToken`/`authorizedAt` derive for free. |
| C9 | Delete friction: type-title-to-confirm? | **Flows wins**: no title typing; checkbox gate only when usage > 0 | Friction proportional to blast radius; typing is a proxy for the comprehension the checkbox provides directly. |
| C10 | `last_error` on status card | **Contract wins**: cut from v1 (M-sized migration; OAuth failures today only reach the analytics sink, `crates/runtara-server/src/product_events.rs:529-533`) | Card is useful without it; UI renders the row only when the prop exists (page-composition's degradation map). |

Cross-cutting decisions adopted without conflict: card replaces both amber banners; stay-on-page save; `AlertDialog` for delete; inline card row (not toast) for test results; pre-opened-popup pattern for any popup after an async step; `resetOptions: { keepDirtyValues: true }` on the `useForm` call before any in-place refetch lands (flows caution #1).
